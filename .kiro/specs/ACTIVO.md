# Spec activo (Kiro)

Este archivo define cuál es el **spec activo** (el que se debe usar como fuente de verdad para ejecutar tasks).

## Cómo usarlo

1. Coloca aquí la **ruta relativa** al directorio del spec activo dentro de `.kiro/specs/`.
2. El spec activo debe contener estos archivos:
   - `requirements.md`
   - `design.md`
   - `tasks.md`

## Ruta del spec activo

Pega una sola línea con la ruta relativa. Ejemplo:

```text
sistema-cotizacion-automatizada
```
